# 05 — SD Card + FatFs (SPI2, Timestamped ADC Logging)

## Overview

Mounts a FAT filesystem on an SD card over SPI2, opens `LOG.TXT` for append,
and writes a timestamped ADC reading once per second. This is a bare-metal
(no RTOS) implementation using HAL blocking SPI and the FatFs middleware
generated by STM32CubeMX.

## Operation Loop

```
1. Mount filesystem          f_mount()
2. Open LOG.TXT (append)     f_open(FA_OPEN_APPEND | FA_WRITE)
3. Loop every 1 second:
     a. Read ADC1 Ch0 (PA0)  HAL_ADC_PollForConversion()
     b. Read uptime counter   HAL_GetTick() / 1000
     c. Format log line       "T=12345s ADC=2048\r\n"
     d. Write to file         f_write()
     e. Flush                 f_sync()
4. On error: blink LED fast and halt
```

`f_sync()` is called every write to minimise data loss on power failure.
For high-frequency logging, call `f_sync()` less often (e.g., every 10 writes).

## CubeMX Configuration

1. **SPI2** — Full-Duplex Master, Prescaler so BAUD < 25 MHz (e.g., /4 = 18 MHz),
   8-bit data, CPOL=0, CPHA=0 (Mode 0). Pins: PB13 SCK, PB14 MISO, PB15 MOSI.
2. **GPIO** — PB12 as GPIO_Output (CS, active-low). Pull HIGH in code before init.
3. **FATFS** middleware — User defined SPI driver. Set USE_LFN=0, _MIN_SS=512.
4. **ADC1** — Channel 0 (PA0), software trigger, single conversion.
5. **SYS** — Timebase: TIM2 (free SysTick for HAL, avoid conflict with FatFs).
6. Clock: 72 MHz.

## SPI Speed Notes

SD card initialisation requires <= 400 kHz. After init the card accepts higher
speeds. The FatFs + STM32 SPI driver generated by CubeMX handles this by setting
the SPI prescaler to /256 during init, then switching to a higher prescaler.
If using a custom driver, implement `SD_SPI_LowSpeed()` / `SD_SPI_HighSpeed()`
functions.

## Expected LOG.TXT Contents

```
T=0s ADC=1024 (0x0400)
T=1s ADC=1031 (0x0407)
T=2s ADC=1028 (0x0404)
```

## Notes

- Code is **not hardware-tested**. Validate on your board.
- FAT32 requires the SD card to be pre-formatted. Use your OS formatter or
  `f_mkfs()` (adds ~5 KB to Flash usage).
- SD cards operate at 3.3 V. Most micro-SD breakout boards include a 3.3 V LDO
  and level shifters for 5 V systems, but check your specific module.
- On Blue Pill (3.3 V system) a simple breakout without level shifting works.
