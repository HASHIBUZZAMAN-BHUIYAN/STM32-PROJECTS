# 01 — FreeRTOS Basics (CMSIS-RTOS v2)

## Overview

Demonstrates FreeRTOS task scheduling on the Blue Pill (STM32F103C8T6) using the
CMSIS-RTOS v2 API generated by STM32CubeMX. Three tasks run concurrently at different
priorities and periods, showing the preemptive scheduler in action.

## Tasks

| Task        | Priority        | Period  | Action                              |
|-------------|-----------------|---------|-------------------------------------|
| LedTask     | osPriorityLow   | 500 ms  | Toggle onboard LED (PC13, active-low) |
| UartTask    | osPriorityNormal| 1000 ms | Send heartbeat string over UART1    |
| SensorTask  | osPriorityHigh  | 200 ms  | Read ADC1 Ch0 (PA0), store result   |

SensorTask runs most frequently and at the highest priority. When it is Ready it
preempts both lower-priority tasks immediately.

## CubeMX Configuration

1. **FreeRTOS** — CMSIS_V2 interface, Total heap size = **8192 bytes**
2. Define three tasks in the FreeRTOS config tab:
   - `LedTask`    — stack 128 words, priority Low
   - `UartTask`   — stack 128 words, priority Normal
   - `SensorTask` — stack 128 words, priority High
3. **ADC1** — Channel 0 (PA0), continuous conversion OFF, software trigger
4. **USART1** — 115200 baud, 8N1, TX=PA9, RX=PA10
5. **GPIO** — PC13 as GPIO_Output (onboard LED)
6. Clock: HSE 8 MHz crystal → PLL → 72 MHz SYSCLK

## Expected Serial Output (115200 baud)

```
[UART] Heartbeat #1
[UART] Heartbeat #2
...
```

ADC readings are stored in `g_adcValue` and can be printed by extending UartTask.

## Notes

- Code is **not hardware-tested**. Validate on your board.
- The onboard LED is active-low: `GPIO_PIN_RESET` turns it ON.
- `osDelay()` yields the CPU; never use `HAL_Delay()` inside FreeRTOS tasks.
