# Project 17: USB CDC Virtual COM Port

## Overview
This project demonstrates the STM32F411's native USB Full-Speed controller as a USB CDC
(Communication Device Class) virtual COM port. ADC1 channel 0 (PA0) is sampled every 500 ms
and the result is transmitted as a human-readable string over USB CDC. Any data received over
USB CDC is echoed back to the host. No external ST-Link or USB-to-UART adapter is needed —
the Black Pill's onboard USB connector handles everything.

## Hardware
- Board: STM32F411CEU6 Black Pill
- MCU clock: 100 MHz (HSE 25 MHz → PLL)
- USB: PA11 (DM), PA12 (DP) — native USB Full-Speed
- ADC input: PA0 (ADC1 CH0, 0–3.3 V analog)
- LED: PC13 (active-low, blinks on each transmission)

## STM32CubeMX Configuration
| Peripheral | Setting |
|---|---|
| USB_OTG_FS | Device Only, no VBUS sensing |
| USB_DEVICE | CDC class |
| ADC1 | IN0, single conversion, SW trigger, 12-bit |
| RCC | HSE 25 MHz, PLL → 100 MHz HCLK, 48 MHz USB clock (PLLQ=5 or PLLQ=4 + 48M from PLLM) |
| SYS | TIM11 as timebase (frees SysTick for USB) |
| PC13 | GPIO_Output, push-pull |

### USB Clock Note
The USB peripheral requires exactly 48 MHz. With HSE=25 MHz:
- PLLM=25, PLLN=192, PLLP=2 (→ 96 MHz then /2 not valid... use PLLN=400, PLLP=4 → 100 MHz HCLK),
- PLLQ=8 → 50 MHz (not valid). Recommended safe config: PLLM=25, PLLN=336, PLLP=4 → 84 MHz
  with PLLQ=7 → 48 MHz. Adjust HAL_RCC_ClockConfig to suit. CubeMX will show a warning if
  48 MHz is not achievable — use 84 MHz system clock in that case.

## DFU Flashing Instructions
1. Hold the **BOOT0** button on the Black Pill
2. While holding BOOT0, briefly press and release **NRST**
3. Wait ~0.5 s, then release BOOT0
4. The board enumerates as **"STM32 BOOTLOADER"** USB device
5. Open **STM32CubeProgrammer**, select USB DFU mode, click Connect
6. Load the `.elf` or `.hex` file and click Download
7. Press **NRST** to exit DFU and run firmware
8. **Alternative:** External ST-Link V2 via SWD — PA13 (SWDIO), PA14 (SWCLK), GND, 3.3 V

## How It Works
1. CubeMX-generated USB CDC middleware is initialised by `MX_USB_DEVICE_Init()`.
2. `main_usercode_init()` configures the ADC.
3. Every 500 ms the main loop starts an ADC conversion, reads the result, converts the raw
   count to voltage (3.3 V reference, 12-bit), formats a string, and calls `CDC_Transmit_FS()`.
4. `CDC_Receive_FS()` callback (in `usbd_cdc_if.c`) sets a global flag and copies received
   bytes; the main loop echoes them back.
5. PC13 LED toggles on each transmission.

## Expected USB Terminal Output (115200 baud or any — CDC ignores baud)
```
ADC: 2048 | Voltage: 1.650V
ADC: 2051 | Voltage: 1.652V
ADC: 1998 | Voltage: 1.608V
```

## Notes
- `CDC_Transmit_FS` returns `USBD_BUSY` if the previous packet has not been sent yet.
  The code retries up to 10 times with a 1 ms delay.
- Do not call `CDC_Transmit_FS` before the USB device is enumerated (wait for
  `hUsbDeviceFS.dev_state == USBD_STATE_CONFIGURED`).
- PA0 should be 0–3.3 V only. Do NOT connect voltages above 3.3 V.
